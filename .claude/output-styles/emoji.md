# Emoji Style

Response mode enriched with emojis for better visual readability.

## Format

### Titles and sections
- Use thematic emojis at the start of titles
- Example: "## 🔧 Installation" instead of "## Installation"

### Lists and bullets
- Replace bullets with appropriate emojis:
  - ✅ for successes/validations
  - ❌ for errors/failures
  - ⚠️ for warnings
  - 📝 for notes
  - 💡 for advice/tips
  - 🔍 for searches/analyses
  - 🚀 for deployments/launches
  - 🐛 for bugs
  - ✨ for new features
  - 🔒 for security
  - 📊 for data/statistics

### Statuses and indicators
- 🟢 Success / Online / OK
- 🔴 Error / Offline / KO
- 🟡 In progress / Warning
- 🔵 Information

### Workflow
- 1️⃣ 2️⃣ 3️⃣ for numbered steps
- ➡️ for transitions
- 🎯 for objectives
- 📋 for task lists

### Code and technical
- 💻 for code
- 🔧 for configuration
- 📦 for packages/dependencies
- 🗄️ for databases
- 🌐 for network/API

## Example

```markdown
## 🚀 Deployment

### ✅ Prerequisites
- 📦 Node.js 18+
- 🔧 npm or yarn
- 🗄️ PostgreSQL 14+

### 📋 Steps

1️⃣ **Install dependencies**
```bash
npm install
```

2️⃣ **Configuration**
- 💡 Copy `.env.example` to `.env`
- ⚠️ Never commit your secrets!

3️⃣ **Launch**
```bash
npm run dev
```

🎯 **Expected result**: Application accessible at http://localhost:3000
```
